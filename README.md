# JT9 Beacon — STM32 HAL

**WORK IN PROGRESS!**

A JT9-1 digital-mode beacon for the STM32F411 (Black Pill), written in C++20
against the STM32 HAL.  The firmware encodes an arbitrary free-text message
into 85 JT9 symbols, drives a **Si5351** clock generator over I²C to frequency-shift
between tones, and displays the current transmit frequency on an **SSD1306 OLED**.
A **rotary encoder** lets you tune the frequency live between transmissions.

---

## Features

- Full JT9-1 DSP pipeline — `packMessage → convolve → interleave → packAndGray → mergeSync` — implemented in pure portable C++, no Arduino or AVR dependencies.
- Symbol timing driven by a hardware timer interrupt (TIM2) for accurate 576.3 ms symbol periods.
- Si5351 HAL port (originally by Jason Milldrum NT7S / Dana H. Myers K6JQ) — all `Wire` / Arduino calls replaced with `HAL_I2C_Master_Transmit` / `HAL_I2C_Master_Receive`.
- SSD1306 OLED display showing the live transmit frequency.
- Rotary encoder with callback-based fluent interface for frequency adjustment and reset.
- TX indicator LED (PC13 on the Black Pill) blinks during transmission.
- CMake + Ninja build with `CMakePresets.json` for Debug / Release configurations.

## Hardware

| Component | Details |
|-----------|---------|
| MCU | STM32F411CEU6 (Black Pill), Cortex-M4 @ 100 MHz |
| RF synthesiser | Si5351A (any breakout), I²C address 0x60 |
| Display | SSD1306 128×64 OLED, I²C |
| Frequency control | Standard incremental quadrature rotary encoder with push-button |
| TX LED | On-board LED, PC13 (active-LOW) |

### Pin assignments

The exact GPIO assignments are generated by STM32CubeMX (see `JT9-beacon-stm32-hal.ioc`).
Key signals:

| Signal | Peripheral |
|--------|-----------|
| I²C1 SDA / SCL | Si5351 + SSD1306 (shared bus) |
| TIM2 | Symbol clock, Update interrupt |
| ENCODER_CLOCK / DATA / SWITCH | Rotary encoder CLK / DT / SW (pull-up inputs) |
| PC13 | TX indicator LED |

## Building

### Prerequisites

- `arm-none-eabi-gcc` (GCC 12+ recommended) on your `PATH`
- CMake ≥ 3.22
- Ninja

On Debian / Ubuntu:
```bash
sudo apt install gcc-arm-none-eabi cmake ninja-build
```

### Configure and build (Debug)

```bash
cmake --preset Debug
cmake --build --preset Debug
```

The output ELF is written to `build/Debug/JT9-beacon-stm32-hal.elf`.

### Release build

```bash
cmake --preset Release
cmake --build --preset Release
```

### Flashing

Use any SWD programmer (ST-Link, J-Link, DAPLink):

```bash
# With OpenOCD
openocd -f interface/stlink.cfg -f target/stm32f4x.cfg \
        -c "program build/Debug/JT9-beacon-stm32-hal.elf verify reset exit"
```

## Configuration

All user-adjustable constants are at the top of `Src/app_main.cpp`:

```cpp
char     wspr_poruka[]   = "YT1GS KN03";  // Message (≤13 chars: A–Z 0–9 space + - . / ?)
uint64_t TX_FREQ_HZ      = 14078600ULL;    // Dial frequency in Hz
uint32_t XO_FREQ_HZ      = 0;             // Reference XO (0 = 25 MHz default)
int32_t  CORRECTION      = 0;             // Frequency correction in ppb
```

### Timer tuning

The symbol period must be ≈ 576.37 ms (JT9-1 standard).  The defaults in
`Beacon.h` target an 84 MHz APB timer clock (STM32F411 at 100 MHz with
standard prescaler settings):

```cpp
#define BEACON_TIM_PRESCALER  8400U   // PSC = 8399 → 10 kHz tick
#define BEACON_TIM_PERIOD     5763U   // ARR = 5762 → 576.3 ms period
```

Override these before including `Beacon.h` if your APB timer clock differs.

## Integration notes

### Timer callback

Forward the HAL timer callback to the `Beacon` instance in `Src/stm32f4xx_it.c`:

```cpp
extern Beacon beacon; // declared in app_main.cpp

void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    beacon.notifySymbolClock(htim);
}
```

### UTC synchronisation

JT9 transmissions must start at the beginning of a UTC minute.  The current
firmware calls `beacon.transmit()` once at startup without waiting for a
minute boundary.  For on-air use, sync the call to a GPS PPS signal or an
RTC alarm aligned to UTC.
